<Title>Star System — Design System</Title>

<Subtitle>Seu guia para criar interfaces consistentes e acessíveis</Subtitle>

O **Star System** é um Design System com temática inspirada no universo, reunindo componentes, tokens e diretrizes que garantem consistência visual, escalabilidade e acessibilidade no desenvolvimento front-end.  
Criado para demonstrar habilidades de desenvolvimento de interfaces, ele utiliza o **Storybook** como ferramenta principal de documentação, garantindo uma visualização clara, exemplos interativos e boas práticas de uso.

<Heading>Princípios</Heading>

- **Consistência** — tipografia, cores e espaçamentos padronizados para uma identidade única.  
- **Escalabilidade** — componentes reutilizáveis e combináveis para múltiplos contextos.  
- **Acessibilidade** — foco, contraste adequado e suporte à navegação por teclado.  
- **Documentação viva** — exemplos, controles e orientações diretamente no Storybook.

<Heading>Tokens</Heading>

- **Cores** — consulte a seção *Colors* para conhecer a paleta oficial.  
- **Tipografia** — fonte padrão *Nunito Sans*, com tamanhos e pesos consistentes.  
- **Espaçamentos e raios** — definidos e aplicados de forma uniforme nos componentes.

<Heading>Componentes</Heading>

- **Atoms** — blocos básicos de UI (ex.: Typography, Radio).  
- **Molecules** — combinações simples de elementos (ex.: Input, Button).  
- **Organisms** — estruturas complexas compostas de múltiplos elementos (ex.: Card).

<Heading>Boas práticas de uso</Heading>

- Utilize sempre as variantes e tamanhos definidos pelos componentes (`variant`, `size`).  
- Explore os controles do Storybook para validar estados e acessibilidade.  
- Evite estilos inline; quando necessário, utilize `className` para ajustes pontuais.
